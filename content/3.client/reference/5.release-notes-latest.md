## Management API auto-publish payload

`createForm` and `patchForm` now accept both:

- `auto_publish`
- `autoPublish`

`autoPublish` is normalized to `auto_publish` in outgoing payloads.

`POST /forms` example:

```ts
await management.createForm({
  title: 'Contact',
  fields: [
    { type: 'text', name: 'full_name', required: true }
  ],
  auto_publish: true
})
```

`PATCH /forms/{key}` example:

```ts
await management.patchForm('contact-form', {
  title: 'Contact v2',
  auto_publish: true
})
```

Without this flag (or when set to `false`), behavior is unchanged and revision remains draft/unpublished.

## Builder publishing behavior

`defaultPublished` no longer triggers a second `/publish` call after save.

Builder save now supports `autoPublish` and sends `auto_publish` directly in create/patch payloads.

Migration impact:

- if your integration expected a separate publish request, update that assumption
- published state can now come directly from create/patch response

## Renderer validation API and external model flow

`FormForgeRenderer` now exposes:

- `validate(options?)`
- `validateField(name)`
- `clearErrors(path?)`
- `getErrors(path?)`

New props:

- `validateOn`
- `validateOnBlur`

In external `v-model` mode, blur-based validation is handled per field (`focusout` path), improving standalone integrations.

## Parent form integration guidance

When embedding renderer data inside a parent form, call `renderer.validate()` before submitting the parent form.

```vue
<script setup lang="ts">
import { z } from 'zod'

const renderer = ref<InstanceType<typeof FormForgeRenderer> | null>(null)
const formData = ref<Record<string, unknown>>({})

const schema = z.object({
  title: z.string().min(1),
  data: z.record(z.string(), z.unknown())
})

async function onSubmitParent(): Promise<void> {
  const result = await renderer.value?.validate()
  if (!result?.valid) {
    return
  }

  // custom submit flow
}
</script>

<template>
  <form @submit.prevent="onSubmitParent">
    <FormForgeRenderer form-key="my-form" v-model="formData" ref="renderer" />
  </form>
</template>
```

Validate embedded FormForge payload as an object (`z.record(...)`), not as `z.string()`.

## Validation strategy

- deterministic rules generated by field type
- `rules(...)` appends custom rules
- `replaceRules(...)` replaces generated rules
- effective schema is resolved before payload validation
- field-only validation is available for isolated input checks without full submission

Nullish behavior depends on `required`:

- `required: false` treats null or empty string as absent
- `required: true` rejects null or empty string

## Submission lifecycle

1. fetch latest or requested version
2. resolve condition-aware effective schema
3. validate payload server-side
4. persist submission
5. run automations (sync or queued, if registered)

## Validate a single field

Use this flow for real-time checks (for example blur/change validation) when you need to validate one input without submitting the whole form.

API signatures:

- `FormInstance::validateField(string $field, mixed $value): array`
- `FormManager::validateField(string $formKey, string $field, mixed $value, ?string $version = null): array`
- `ScopedFormManager::validateField(string $formKey, string $field, mixed $value, ?string $version = null): array`

Valid example:

```php
<?php

declare(strict_types=1);

use EvanSchleret\FormForge\Facades\Form;

$result = Form::get('contact')->validateField('email', 'john@example.com');

if ($result['valid']) {
    $validatedEmail = $result['validated']['email'] ?? null;
}
```

Invalid example:

```php
<?php

declare(strict_types=1);

use EvanSchleret\FormForge\Facades\Form;

$result = Form::get('contact')->validateField('email', 'not-an-email');

if (! $result['valid']) {
    $errors = $result['errors'];
}
```

Response payload shape:

- `valid` (`bool`)
- `errors` (`array`)
- `validated` (`array`)

Edge cases:

- unknown field: validation fails with `valid: false` and a field error in `errors`
- unknown form or unknown version: not-found response from form/version resolver

## Field validation config

`validation.field.stop_on_first_failure` (bool, default: `false`)

- `false`: returns all rule errors collected for the target field
- `true`: stops at first failing rule and returns a single first error
